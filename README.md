# CarFinder

## What did you build and why?
CarFinder is an AI-powered car search platform tailored for the Indian market. The traditional car buying experience often involves complex filters and overwhelming specs. I built CarFinder to allow users to search for cars using natural language . By integrating a conversational AI interface, users can describe their life situations, budgets, and preferences as if they were talking to an expert, making the car discovery process seamless and intuitive.

**What did you deliberately cut?**
- **User Authentication & Profiles:** Dropped to focus purely on the core AI search experience and keep the initial MVP lean.
- **Complex UI Filters:** Deliberately replaced traditional dropdowns/sliders with a unified chat interface to force the reliance on natural language processing.


## What’s your tech stack and why did you pick it?
- **Frontend:** React 19, Vite, and Tailwind CSS. Chosen for rapid prototyping, component-based architecture, and highly responsive styling without the overhead of heavy CSS frameworks.
- **Backend:** Node.js and Express.js. Perfect for building a lightweight, non-blocking API that interfaces seamlessly with the frontend.
- **Database:** MongoDB (via Mongoose). Ideal for handling semi-structured car data and executing dynamic queries generated by the LLM.
- **AI / LLM:** Groq SDK. Chosen for its extremely fast inference speeds, which is critical for real-time conversational search experiences.

## What did you delegate to AI tools vs. do manually? 
- **Delegated to AI tools:** 
  - Boilerplate generation for the React frontend and Express backend.
  - Tailwind CSS styling for the landing page and chat interface.
  - Crafting the highly complex LLM system prompt (`Prompt.js`) that converts natural language, 
- **Done manually:** 
  - Integrating the Groq API and ensuring it reliably returns properly formatted JSON.
  - The scoring algorithm (`scoreCar`) to rank the cars based on LLM-extracted preferences.
  - Overall system architecture and routing logic between the React frontend and the Express backend.

**Where did the tools help most?**
antigravity was really helpful in generating the complex prompt,  handling the nuanced variations of user queries (like "10L", "family car"), and rapidly spinning up the UI components (like the landing page).

**Where did they get in the way?**

I researched how people actually search for cars and noticed a clear pattern: traditional filter UIs are exhausting. Users have to pick a brand, choose a body type, set a price range, scroll endlessly through listings, and start over if nothing fits. Most people don't shop that way — they think in situations and needs, not spec sheets.
CarFinder replaces that heavy filter grind with a single chat interface, making car search faster and effortless for the customer.

## If you had another 4 hours, what would you add?
- Rich Car Profiles & AI Reviews — Expand each result into a full detail page with high-resolution images, complete specs, and an AI-generated review explaining why that car fits the user's specific needs. Users can save favorites and revisit their full chat history anytime.

- **More Robust Error Handling:** Better fallback UI/UX when the LLM fails to generate a valid JSON query or misunderstands a prompt.
- **Detailed Car Views:** Expand the UI to show high-resolution images, full specs, and financing options.
